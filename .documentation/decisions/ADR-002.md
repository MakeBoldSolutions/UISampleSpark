# ADR-002: OpenAPI Tooling Migration — Swashbuckle to ApiTestSpark

## Status

Accepted

## Context

`Swashbuckle.AspNetCore` was the de-facto OpenAPI/Swagger library for ASP.NET Core but was
not updated to support .NET 9/10 built-in OpenAPI support, leading to compatibility warnings
and CI Trivy vulnerability flags. ASP.NET Core 9+ ships `Microsoft.AspNetCore.OpenApi` natively,
making third-party Swagger middleware largely redundant.

## Decision

Replaced `Swashbuckle.AspNetCore` with:

- `Microsoft.AspNetCore.OpenApi` (built-in, maintained by Microsoft)
- `ApiTestSpark` (lightweight UI layer over the native OpenAPI document endpoint)

The OpenAPI document is served at `/openapi/v1.json`; `ApiTestSpark` renders the interactive
explorer UI without the Swashbuckle middleware stack.

## Consequences

### Positive

- Eliminates CVE exposure from unmaintained Swashbuckle releases
- Reduces package dependency footprint by removing `Swashbuckle.AspNetCore.SwaggerGen` and `SwaggerUI`
- Uses the same `IOpenApiDocument` API that ships with the .NET runtime
- CI Trivy scans no longer flag Swashbuckle as a vulnerable dependency

### Negative

- `ApiTestSpark` is a newer, less battle-tested library; edge cases in OpenAPI schema generation
  may surface that Swashbuckle handled automatically
- Some Swashbuckle operation filter patterns need to be rewritten as `IOpenApiOperationTransformer`

# ADR-003: Minimal API Migration in UISampleSpark.UI Host

## Status

Accepted

## Context

The `UISampleSpark.UI` host exposed its REST API endpoints through MVC controllers
(`[ApiController]` + routing attributes). While functional, the controller-based approach
added ceremony (action filters, model binding pipeline, controller base class) for a simple
data API whose consumers are the front-end demo pages served by the same host.

## Decision

Replaced the controller-based REST API in `UISampleSpark.UI` with ASP.NET Core Minimal API
endpoints registered directly in `Program.cs` (or via `IEndpointRouteBuilder` extension methods).
The existing `IEmployeeService` / repository interfaces remain the authoritative data access layer;
only the HTTP routing surface changed.

## Consequences

### Positive

- Eliminates `ControllerBase` and `[ApiController]` boilerplate for simple CRUD endpoints
- Demonstrates the Minimal API pattern as an additional educational front-end alongside MVC,
  Razor Pages, Blazor, React, Vue, htmx, and Vanilla JS
- Reduces middleware pipeline overhead for API routes
- Consistent with the `UISampleSpark.MinimalApi` standalone project that already uses this pattern

### Negative

- Loses built-in MVC model validation pipeline (`ModelState`); manual validation must be wired
  where required
- Controller-level authorization attributes become inline `RequireAuthorization()` calls,
  which is less discoverable for developers accustomed to MVC patterns
